# Plan: Find-in-Page Search for emede

## Overview

Add a text search tool to emede's markdown viewer. Since Tauri v2 does not expose a native find-in-page API, implement it as a pure JavaScript DOM-based solution that wraps matches in `<mark>` elements and provides next/prev navigation.

## Design

### Search Bar UI (pattern follows settings panel / about overlay)

- Position: Fixed bar across the top of the reader area (below titlebar)
- Contains: text input, match counter (`1/10`), prev/next buttons, close button
- Toggled via `Ctrl+F` / `Cmd+F` (universal shortcut, checked before mode-specific handlers)
- `Escape` closes search bar + clears highlights
- `Enter` in input = next match, `Shift+Enter` = prev match
- Auto-focuses input when opened
- Respects same `.hidden` + `aria-hidden` toggle pattern as other panels

### FindInPage class (`src/find.js`)

```
class FindInPage {
  constructor(container)    // contentEl as container
  find(query)               // walk text nodes, wrap matches in <mark>, return count
  next()                    // scroll to next match, update active index
  prev()                    // scroll to previous match
  stop()                    // remove all <mark> tags, restore text nodes
  get activeIndex()         // 0-based position of current match
  get matchCount()          // total matches
}
```

Implementation details:
- Walk text nodes inside `container` using a `TreeWalker` with `NodeFilter.SHOW_TEXT`
- Skip text nodes inside `<script>`, `<style>`, `<pre>`, `<code>` (code blocks have their own markup)
- For each match, split the text node and wrap matches in `<mark data-find-match="">`
- Active match gets `.find-active` class for distinct highlighting
- `scrollIntoView({ block: 'center' })` on active match
- `stop()` replaces each `<mark>` with its text content, restoring original DOM

### Styles

### Keybinding Changes

Add to `handleAppShortcuts`:
- `Ctrl+F` (or `Cmd+F` via `event.metaKey`): toggle search bar open

**Conflict resolution**: In Vim mode, `Ctrl+F` was previously page-down. Since search is a fundamental feature and `Ctrl+F` is the universal standard, override it. Vim mode still has `Space`, `f`, and `Ctrl+d` for page-down.

Add `Ctrl+F` to all keybinding help tables.
